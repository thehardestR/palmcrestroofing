# Palm Crest Roofing — Website

Static website for **Palm Crest Roofing**, the Los Angeles County brand from the team behind Burbank Roofing (CA Lic. #1109337).

## Quick facts

- **Domain:** palmcrestroofing.com
- **Stack:** Static HTML / CSS / JS — no framework, no build step for the hub pages
- **Hosting:** GitHub Pages (CNAME configured)
- **Font:** Montserrat (Google Fonts)
- **Forms:** Formspree endpoint `xeezrwvn` (shared with Burbank Roofing — same inbox). Leads are tagged by `source_domain` hidden field in `js/main.js` so you can filter Palm Crest leads from Burbank leads.

## Building

The 7 hub pages (index, about, contact, faqs, reviews, gallery, free-inspection) are **hand-authored** — edit the HTML directly.

The 12 service pages, 20 city pages, and the areas index are **generated** from JSON configs. After editing anything in `build/services/*.json` or `build/areas/*.json`:

```bash
cd palmcrest
node build/build.js
```

This regenerates:
- `services/<slug>.html` for each service JSON
- `areas/<slug>/index.html` for each area JSON
- `areas/index.html` (city directory grouped by cluster)
- `sitemap.xml`

No external dependencies — just Node.js ≥ 14.

## Adding a new city

1. Create `build/areas/<slug>.json`:
   ```json
   {
     "slug": "city-slug",
     "city": "City Name",
     "cluster": "San Fernando Valley",
     "short_blurb": "One-line description.",
     "neighborhoods": ["Neighborhood 1", "Neighborhood 2"],
     "local_note": "2–4 sentences of city-specific roofing context.",
     "faqs": [
       { "q": "...", "a": "..." }
     ]
   }
   ```
2. Run `node build/build.js`.

Cluster options: `San Fernando Valley`, `Santa Clarita Valley & North County`, `San Gabriel Valley`, `Westside & Beaches`, `South Bay & Long Beach`, `Southeast LA County`. New clusters can be added in `build/build.js` in the `buildAreasIndex` function.

## Adding a new service

1. Create `build/services/<slug>.json` following the pattern of existing files (see `residential-roofing.json` for the full schema).
2. Run `node build/build.js`.
3. Add the service card to `index.html` and the footer service lists.
